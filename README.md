# Работа по курсу Цифровые архивы 2024

Этот репозиторий содержит коллекцию веб-архивов кинофестивальных сайтов, созданную с использованием инструмента [`wpull`](https://wpull.readthedocs.io/en/master/) для загрузки файлов, описание метаданных полученное с помощью [`metawarc`](https://github.com/datacoon/metawarc), а также результаты анализа коллекции.

## Краткое описание

В данной работе мы сохраняли архивы сайтов российских кинофестивалей, объединенных общей тематикой: фестивали, представляющие современное российское и международное кино. 

Коллекция веб-архивов кинофестивалей – набор ресурсов в открытом доступе, содержащих информацию о программах, участниках, организаторах и истории проведения киносмотров. Данные материалы представляют ценность для культурологических исследований, изучения истории кино и анализа культурной политики.

В коллекцию входят:
- [Международный кинофестиваль "17 моментов"](https://www.kinofest17.tv/)
- [Международный кинофестиваль "Окно в Европу"](https://oknofest.com/) 
- [Международный кинофестиваль "Зеркало" имени Андрея Тарковского](https://zerkalo.space/)

Собранные данные и материалы могут быть использованы для:
- Исследования визуальной культуры и дизайна кинофестивальных сайтов
- Анализа программ фестивалей и участников
- Сохранения цифровой истории культурных событий
- Академических исследований в области киноведения и культурологии

Структура проекта
Проект организован в виде отдельных папок для каждого кинофестиваля:

text
project/
├── 17_moments/
│   ├── README.md              # Детальный анализ архива kinofest17.tv
│   ├── analysis_17moment.txt  # Результат команды analyze
│   └── digital_meta_17moments.jsonl  # Результат команды metadata
├── okno/
│   ├── README.md              # Анализ архива oknofest.com
│   └── analysis_okno.txt      # Результат команды analyze
└── zerkalo/
    ├── README.md              # Анализ архива zerkalo.space
    └── analysis_mmkf.txt      # Результат команды analyze

Каждая папка содержит markdown-файл с подробным анализом соответствующего архива и текстовый файл с результатами статистического анализа. Для архива "17 моментов" дополнительно сохранен файл в формате JSONL с полными метаданными, извлеченными командой metawarc metadata.

## Метод и технология

В данной работе мы использовали несколько инструментов для создания и анализа коллекции веб-архивов.

### 1. Архивация сайтов

Для сохранения архивов использовался инструмент `wpull` с базовыми параметрами:
```
wpull --warc-file [имя_архива] --recursive --level=5 [URL_сайта]
```

### 2. Анализ метаданных

Для каждого архива производился анализ метаданных с помощью `metawarc`:

**Для успешного архива (kinofest17.tv):**
```
metawarc analyze 17moments_archive_full.warc.gz > analysis_17moment.txt
metawarc metadata --output digital_meta_17moments.jsonl 17moments_archive_full.warc.gz
```

**Для проблемных архивов (oknofest.com, zerkalo.space):**
```
metawarc analyze okno.warc.gz > analysis_okno.txt
metawarc analyze mmkf.warc.gz > analysis_mmkf.txt
```

*Примечание:* Команда `metawarc metadata` для архивов oknofest.com и zerkalo.space не сработала, что указывает на технические проблемы с этими архивами.

### 3. Результаты анализа

### Успешный кейс: kinofest17.tv
- **Объем:** 765 МБ (сжатый формат .warc.gz)
- **Контент:** Полноценный медиаархив с изображениями (91% от общего объема), документами и структурой сайта
- **Качество:** Сохранены метаданные изображений (EXIF), PDF-документы, динамически генерируемый контент

### Проблемные кейсы: oknofest.com и zerkalo.space
- **Объемы:** 132 КБ и 4 МБ соответственно
- **Контент:** Только HTML-страницы без медиафайлов и стилевого оформления
- **Проблемы:** 
  - Неполнота архивов (отсутствие изображений, CSS, JavaScript)
  - Технические проблемы с извлечением метаданных
  - Ограниченная функциональность сохраненных версий

## Итог работы

В ходе работы мы создали коллекцию веб-архивов трех российских кинофестивалей, обращая внимание на технические аспекты архивации и анализа данных. Важными результатами работы стали:

### Технические выводы:
1. **Разнородность качества архивации** - от полного медиаархива (kinofest17.tv) до минимальных текстовых версий (oknofest.com, zerkalo.space)
2. **Влияние архитектуры сайтов** на успешность архивации (динамические сайты с JavaScript сложнее архивировать)
3. **Важность корректных настроек архиватора** для загрузки внешних ресурсов и медиафайлов

### Содержательные выводы:
1. **Культурная ценность** успешно архивированных медиаматериалов (афиши, фотографии, документация фестивалей)
2. **Проблема цифрового вымирания** - сайты с современными технологиями оказываются более уязвимыми для потери контента при архивации
3. **Необходимость адаптации методов архивации** под конкретные типы сайтов и используемые технологии

Работа подчеркнула важность внимательного подхода к выбору инструментов архивации и необходимости формулирования четких целей и требований перед началом работы с цифровыми архивами культурного наследия.
